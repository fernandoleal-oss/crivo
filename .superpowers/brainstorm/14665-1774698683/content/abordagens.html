Como integrar Desirre e Fabi no Crivo?

Três abordagens — escolha a que faz mais sentido pro demo

A

Persona swap + features novas

Substitui Fernanda → Desirre e Marcos → Fabi no RoleSwitcher. Cada role ganha features que resolvem a dor real delas:

  • Desirre (Atendimento): ao criar projeto, aparece um "Conferência de Briefing" — checklist inteligente que identifica o que falta (verba, prazo, assets, aprovação interna)
  • Fabi (Mídia): no claquete digital, botão "Exportar para veiculação" gera os mesmos dados em formatos diferentes (PDF cliente, planilha interna, TV spec)

⭐ Recomendado — menor esforço, maior impacto no demo

B

Features standalone dentro do projeto

Mantém os nomes genéricos, mas cria abas/seções novas no projeto:

  • Aba "Briefing": formulário estruturado com campos obrigatórios + IA que pontua completude (0–100%) e lista o que falta
  • Aba "Mídia": form único de dados de veiculação que gera exports em 3 formatos diferentes automaticamente

Mais robusto, mas demanda mais build

C

Contextual — avisos proativos por role

O app fica "mais esperto" para cada role, sem telas novas:

  • Desirre: ao abrir projeto, um banner âmbar alerta "Briefing incompleto — faltam 3 campos" com link para completar
  • Fabi: ao editar claquete, tooltip/badge mostra quais campos ainda vão para qual formato de export

Sutil — bom como complemento, fraco como feature principal
